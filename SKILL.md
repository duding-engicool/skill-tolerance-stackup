# 公差叠加与尺寸链计算技能

## 一、角色定位（给谁用）

本技能服务于需要做**公差叠加 / 尺寸链（Tolerance Stack-up / Dimension Chain）**计算的工程师：

- **尺寸工程师**：在总成设计中验证各零件尺寸累积后是否满足装配间隙/干涉要求。
- **工艺工程师**：在工艺规划阶段评估工序尺寸链，判断累积公差是否可控、是否需要收紧某环。
- **设计工程师**：在设计阶段做公差分配（把总公差合理分到各组成环）。

**岗位画像（典型）**：用户面对一个装配体，手里有一串组成环尺寸及各自公差（有的增环、有的减环），担心"累加起来会不会超差、装不上或晃量大"。用户要的是**算清楚累积公差、给出 WC 与 RSS 两种结果、判断是否满足间隙要求**，而不是一段尺寸链理论课。

> 本技能做**一维线性尺寸链**的解析计算（含 WC 极值法与 RSS 统计法），不替代三维公差分析软件（如 VisVSA/Sigmetrix）与有限元分析。

## 二、使用场景（工作环节 / 触发条件 / 手头材料）

| 场景 | 触发条件 | 用户手头通常有的材料 |
|------|----------|----------------------|
| 装配可行性验证 | 设计定型前/后验证间隙/干涉 | 各组成环基本尺寸+公差、装配间隙要求 |
| 公差分配 | 总公差给定，需分解到各环 | 总公差、环数量、工艺能力 |
| 超差根因定位 | 装配超差，找是哪环贡献大 | 实测尺寸、各环公差、链结构 |
| 工艺尺寸链 | 工序尺寸累积是否超差 | 工序尺寸链、工序公差 |
| 与 GD&T 衔接 | GD&T 标注核对后做叠加验证 | gdt-checker 输出的位置度等 |

**激活触发词**：公差叠加、尺寸链、累积公差、极值法、WC、统计法、RSS、间隙计算、干涉分析、公差分配、装配公差、位置度叠加。

## 三、解决的问题（业务痛点，非动作）

1. **手工计算易错**：多环相加、正负号、增环减环判断靠手算，环多时极易算错。
2. **装配累积超差**：单件都合格，但累积后间隙不够/干涉/晃量大，装机才暴露。
3. **方法选择困惑**：不知该用极值法（保守）还是 RSS（统计，更经济），导致要么过度收紧要么冒险。
4. **超差责任不清**：累积超差后不知道该收紧哪一环、收紧多少。
5. **与 GD&T 脱节**：位置度等几何公差如何纳入线性叠加，缺乏统一处理。

## 四、能力边界（能做 / 不做 / 输入要求）

**能做**：
- 区分增环/减环，计算封闭环基本尺寸、极值法（WC）公差、RSS 统计公差。
- 对比装配间隙/干涉要求，给出"满足/不满足"及超差裕度。
- 给出简单公差分配建议与超差贡献提示。
- 生成**尺寸链计算报告**（纯文字版 .txt + Markdown .md，含计算逻辑），脚本内含数值计算函数。

**不做**：
- 不替代三维公差分析软件（VisVSA 等）做复杂空间/非线性叠加。
- 不替代有限元/运动仿真。
- 不替代设计对公差的最终拍板；计算结果为建议，需结合工艺能力（Cpk）判断。
- 不编造材料特性/分布假设；若假设非正态等，标「待企业补充」。

**输入要求**：
- 至少提供：各组成环基本尺寸 + 公差（±值或上下偏差），并标明增环/减环（或给链方向由技能判断）。
- 提供装配间隙要求（间隙下限/上限或干涉限制）。缺失则标「待补充」并追问。

## 五、交互与角色设定（语气 / 采集方式 / 是否强制交互锁 / 引导深度）

- **语气**：专业、计算严谨、像一位尺寸工程老手；给出公式与代入过程，让用户可复核。
- **采集方式**：先确认"链结构（增环/减环）、各环尺寸公差、装配要求"，再计算；提供表格/截图也可直接读。
- **强制交互锁**：默认**不强锁**。信息足够则直接算；缺装配要求时给出 WC/RSS 结果但标"是否满足待补充要求"，不阻断计算。
- **引导深度**：深。给出两种方法的公式、代入、对比与超差提示，必要时给分配建议。

## 六、处理流程

```
步骤1 建立尺寸链
  └─ 确定封闭环（装配要求），画出链，标各组成环

步骤2 区分增环/减环
  └─ 按对封闭环的影响方向判定（增环+/减环-）

步骤3 极值法（WC）
  └─ 封闭环基本尺寸 = Σ增 - Σ减
  └─ WC 公差 = Σ|各环公差|

步骤4 统计法（RSS）
  └─ RSS 公差 = √(Σ 各环公差²)   （假设各环独立、近似正态）

步骤5 对比装配要求
  └─ 间隙 = 封闭环尺寸；判断 WC/RSS 下是否满足间隙/干涉限制

步骤6 超差提示与分配建议
  └─ 若超差，指出贡献最大环、建议收紧方向

步骤7 生成报告
  └─ 调用 scripts/build_report.py 渲染纯文字版 .txt + Markdown .md（含计算逻辑）
```

## 七、输入输出（格式 + 示例）

**输入（结构化）**：

```json
{
  "title": "盖板与壳体装配间隙",
  "date": "2026-07-13",
  "closed_loop": "装配间隙 G",
  "requirement": {"type": "间隙", "min": 0.10, "max": 0.60},
  "chains": [
    {"name": "壳体高度 H1", "nominal": 50.0, "tol": 0.20, "dir": "增环"},
    {"name": "垫片厚 T",  "nominal": 2.0,  "tol": 0.05, "dir": "增环"},
    {"name": "盖板高 H2",  "nominal": 51.5, "tol": 0.15, "dir": "减环"}
  ],
  "gaps": []
}
```

**输出**：尺寸链计算报告——封闭环基本尺寸、WC 与 RSS 公差、间隙分布、是否满足要求、超差提示（纯文字版 .txt + Markdown .md）。

**计算逻辑（脚本内含）**：
- 封闭环基本尺寸 `N = Σ(增环 nominal) − Σ(减环 nominal)`
- WC 公差 `T_wc = Σ|tol_i|`
- RSS 公差 `T_rss = sqrt(Σ tol_i²)`
- 间隙范围：`[N − T/2, N + T/2]`（对称假设下）

## 八、内置知识 / 参考清单

**1. 极值法（Worst Case, WC）**
- 假设所有组成环同时取最不利极限，结果最保守。
- 封闭环公差 = 各组成环公差绝对值之和：`T_WC = Σ|Ti|`。
- 适用：安全性/法规件、环数少、要求 100% 装配保证。

**2. 统计平方和方法（Root Sum Square, RSS）**
- 假设各环独立、近似正态，公差按统计合成。
- `T_RSS = √(T1² + T2² + … + Tn²)`。
- 适用：大批量、环数多、允许一定装配合格率（非 100%）。

**3. 增环/减环判定**
- 增环：该环尺寸增大使封闭环增大；减环：使封闭环减小。
- 封闭环基本尺寸 = Σ增环 − Σ减环。

**4. 公差分配（反向）**
- 等公差法：`Ti = T_total / n`（WC）或 `Ti = T_total / √n`（RSS，近似）。
- 按工艺能力加权分配：难加工环给大公差，易保证环给小公差。

**5. 与 GD&T 衔接**
- 位置度等几何公差转化为线性叠加时，按最大实体（MMC）边界计入，或按 ±(位置度/2) 线性近似（「待企业补充」具体转化规则以企业标准为准）。

> 公式与系数（如 6σ 对应 Cpk 等）为通用工程知识；分布假设非正态时标「待企业补充」。
